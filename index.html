GIT

Version Control

Local

You don't have to be connected to make commits.

Fast

It only stores delta changes, which makes it fast and space efficient.

Safe

You can go back an forth between commits instantaneously.

Branching

Built around branching and merging. Makes conflicts in large projects managable.

How does it look?

Short tutorial


git init // initialize
git add Renderer.h // adds a local change
git add . //or add all local changes automatically
git commit -m "Added a renderer" // locally commits the change
git remote add origin <server> // sets the server (only once)
git push origin master // pushes the change to the server

Git Bisect

Incredible tool that helps you to track down errors

  • Remeber to commit often
  • Make sure that your commits have no build errors

How does it look in real life?

  • Fork a repository // makes it your own (Similar to branching)
  • Make some changes locally
  • Push those changes to your repository
  • Create a pull request (merge your changes)

How can I make sure that a pull request actually compiles?

Answer: Continous integration

  • External server will pull the pull request
  • It will run the build script and it will execute the tests
  • Shows you the output
  • Everything happens automatically
